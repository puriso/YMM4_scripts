# insert_gap_layer

YMM4 MCP を使って、指定レイヤー上のセリフアイテム同士が詰まりすぎないように後続アイテムを右へ移動する補助スクリプトです。

## 目的

- 指定レイヤーのセリフアイテムをフレーム順に並べます。
- 前のセリフ末尾が `。！？` の場合だけ、次のセリフとの間に指定フレーム数の空白を作ります。
- 句読点条件に当たらない場合でも、セリフ同士が重なっているときは重なりを解消します。
- 変更がある場合は YMM4 に反映し、そのままプロジェクトを保存します。

## 前提条件

- Python 3
- YukkuriMovieMaker4
- YMM4 MCP 連携サーバー

外部 Python パッケージは使っていません。

通常は YMM4 と MCP 連携サーバーを起動し、対象プロジェクトを開いた状態で実行します。接続先の既定値は `http://localhost:8765/api` です。

## 使い方

リポジトリのルートディレクトリから実行します。

```powershell
python scripts\insert_gap_layer\insert_gap_layer.py
```

実行すると、変更内容を表示してから YMM4 へ反映し、プロジェクトも保存します。

## 主なオプション

- `--base-url`
  - YMM4 MCP API の URL。
  - 既定値: `http://localhost:8765/api`
- `--layer`
  - 対象レイヤー。
  - 既定値: `01`
  - `01`、`1`、`02`、`2` のように指定できます。
- `--gap`
  - セリフ間に入れる空白フレーム数。
  - 既定値: `12`
詳しい引数は `--help` でも確認できます。

```powershell
python scripts\insert_gap_layer\insert_gap_layer.py --help
```

## gap 判定仕様

前のセリフの末尾を見て gap を入れるかを判定します。

前後の空白や改行は無視します。末尾が閉じかぎ括弧 `」` の場合は、その 1 文字前で判定します。

例:

- `今日は速い。` は gap を入れます。
- `今日は速い！` は gap を入れます。
- `今日は速い？` は gap を入れます。
- `「今日は速い。」` は gap を入れます。
- `「今日は速い」` は gap を入れません。
- `今日は速い` は gap を入れません。

## 変更内容の考え方

このスクリプトは、対象レイヤーのセリフを先頭から順番に見ます。

次のセリフの開始フレームが必要位置より前にある場合だけ、そのセリフを右へ移動します。必要位置は次のどちらかです。

- gap を入れる場合: `前のセリフ開始フレーム + 前のセリフ長 + gap`
- gap を入れない場合: `前のセリフ開始フレーム + 前のセリフ長`

一度移動した位置は後続セリフの計算にも反映します。

## 安全性

- 変更対象がある場合は、対象レイヤー、gap 設定、移動前後のフレームを表示します。
- 変更対象がある場合は、YMM4 へ反映してからプロジェクトを保存します。
- MCP サーバーに接続できない場合は、YMM4 と MCP 連携サーバーの起動確認を促すエラーを表示します。

## 実行例

レイヤー 02 を対象にして、24 フレームの空白を入れる場合:

```powershell
python scripts\insert_gap_layer\insert_gap_layer.py --layer 02 --gap 24
```
